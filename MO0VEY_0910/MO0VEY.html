Github Basic & Most Important Local & Remote Commands

  • git init [Local]
    Initializes a new Git repository in your current directory. This creates a hidden .git folder that tracks your project’s version history.
    git init
  • git status [Local]
    Shows the current state of the working directory and staging area. It lets you see which changes have been staged, which haven’t, and which files aren’t being tracked by Git.
    git status
  • git add [Local]
    Stages changes (new, modified, or deleted files) to be included in the next commit. You can add specific files or all changes.
    git add filename.txt
    git add .
  • git commit [Local]
    Saves the staged changes to the local repository with a descriptive message. Commits are snapshots of your project history.
    git commit -m "Describe your changes"
  • git remote add [Local/Remote]
    Adds a remote repository (usually on GitHub) so you can push and pull changes. The remote is given a name (commonly origin).
    git remote add origin https://github.com/username/repo.git
  • git pull [Remote]
    Fetches changes from a remote repository and merges them into your local branch. Keeps your local code up to date with others.
    git pull origin main
  • git push [Remote]
    Uploads your local commits to a remote repository (like GitHub), making them available to others.
    git push origin main

Used up Extensions

  • Prettier - Code formatter
    Prettier is a reliable tool for maintaining clean and consistent code formatting. It automatically organizes your files on save, ensuring that indentation, spacing, and quotation marks follow best practices across different languages.
  • Live Server
    Live Server provides a straightforward way to preview your web projects. By launching a local server, it refreshes your browser instantly whenever you update your HTML, CSS, or JavaScript, streamlining the development workflow.
  • ESLint
    ESLint is an essential extension for JavaScript development. It scans your code for errors and enforces coding standards, helping you catch issues early and maintain a high level of code quality throughout your project.
  • HTML CSS Support
    This extension enhances productivity by providing intelligent autocomplete and validation for CSS classes and IDs within HTML files. It helps reduce mistakes and speeds up the process of styling web pages.
  • GitLens — Git supercharged
    GitLens extends the built-in Git capabilities of VS Code. It offers detailed insights into code changes, including author information and revision history, making it easier to track progress and collaborate effectively.
